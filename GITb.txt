//Как скопировать на свой компьютер репозиторий из github.com, что должно быть сделано предварительно?


Создать локальную папку на своем компьютере, в которую будет скопирован репозиторий.

Перейти на страницу репозитория на Github, который вы хотите скопировать.

Нажать на кнопку "Code" и выбрать вкладку "HTTPS". Скопировать URL-адрес репозитория.

Открыть командную строку или терминал на своем компьютере.

Ввести команду git clone и вставить скопированный ранее URL-адрес репозитория


========================================================================================================================

//Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?

Открыть терминал 

перейти в каталог, который  хотите сделать Git репозиторием

Инициализируйте Git репозиторий в выбранном каталоге, выполнив команду git init. 

Добавить файлы в Git репозиторий, используя команду git add.

Создать коммит с помощью команды git commit.

Создайте новый репозиторий на Github.com.

Скопируйте URL-адрес вашего нового репозитория на Github.com.

Добавьте ссылку на удаленный репозиторий с помощью команды git remote add origin

Отправьте ваш репозиторий на Github.com, используя команду git push. 

========================================================================================================================

//Как получить на локальный компьютер изменения с github.com

Откройте терминал или командную строку на своем компьютере и перейдите в каталог, в котором находится ваш локальный репозиторий.

Выполните команду git fetch. 

ыполните команду git merge origin/master.

==========================================================================================================================

//Какую информацию нам дают команда :
Команда git status показывает текущее состояние репозитория Git, включая изменения, которые были произведены в файловой системе, но ещё не добавлены в индекс для коммита.

Команда git log отображает историю коммитов в репозитории Git. Она показывает список коммитов в обратном хронологическом порядке (начиная с последнего коммита) и содержит информацию о каждом коммите, включая его хэш-код (или идентификатор), автора, дату и время коммита, а также сообщение коммита.

Команда git remote -v показывает список удаленных репозиториев, связанных с локальным репозиторием.
